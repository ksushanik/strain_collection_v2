# План адаптации под мобильные устройства

Ветка: `feat/mobile-responsive`  
Цель: обеспечить полноценную адаптивность (телефоны ≥320px, планшеты ≥768px) для основных экранов.

## Обзор целей
- Критические страницы: список/карточка штаммов, формы создания/редактирования штаммов и образцов, storage-view (ячейки), медиа, настройки (ui-bindings), логин.
- Компоненты: формы (strain-form, sample-form), списки (strain-list, sample-list), photo-upload/strain-photo-upload, storage-view.
- Навигация/хедер: убедиться в корректной работе меню/таблиц/кнопок на маленьких ширинах.

## Этапы работы
- [ ] Аудит текущих экранов на 320–768px: выявить ломания, скролл, переполнения, размеры таблиц/кнопок/форм.
- [ ] Разработать сетку и токены: контейнеры, отступы, скрытие колонок, стек блоков на мобильных.
- [ ] Реализовать адаптивность по шагам:
  - [ ] Списки (штаммы/образцы/медиа): стеки вместо таблиц, скрытие второстепенных колонок, компактные фильтры.
  - [ ] Формы (штамм/образец): одно-колоночная сетка на <768px, responsive inputs/buttons.
  - [ ] Storage view: responsive карточки боксов и ячеек (скролл/гриды), упрощённый инфо-блок.
  - [ ] Фото-аплоады: гибкие превью, управление кнопками в одну колонку.
  - [ ] Логин/настройки: выравнивание и отступы под мобильный.
- [ ] Тестирование: manual на 320/375/768 + `npm run lint` + `npm run build` (frontend).

## Проверки перед завершением
- [ ] Линт (`npm run lint` в frontend).
- [ ] Сборка (`npm run build` в frontend).
- [ ] Спот-проверка экранов на 320/375/768: формы, списки, storage, аплоады, навигация/хедер.
