# Legend

- **Назначение**: текстовая легенда/справка для UI (условные обозначения, цветовые коды, сокращения).
- **API**: `GET /settings/legend`, `PUT /settings/legend` (тело `{ content: string }`).
- **Фронтенд**:
  - `/legend` — просмотр/редактирование глобального текста (Sidebar).
  - `/settings` — для каждого UI binding можно задать Legend override (заполняется в таблице секций).
  - `/dynamic/storage` отображает легенду, если она задана у соответствующего binding (или будет показан фолбэк, если добавим).
- **Кто правит**: администраторы/менеджеры. Остальные могут читать (ролевая модель CASL позволяет read Legend).
- **Использование**: можно выводить выдержки на страницах (пока только отдельная страница). При фичах, добавляющих новые обозначения, обновлять content.
